Navigation Concepts

5 options that match the diff view hero aesthetic. Pick one.

Concept 1 — GitHub Underline Tab Bar
Full-width flat bar with bottom underline indicator
How GitHub's repo tabs work. No container, no pill. Just text on the page with a colored underline that slides between items. The nav IS the page — no visual separation.

Concept 2 — VS Code Title Bar
Compact tab strip like a code editor
Looks like VS Code's tab bar. Dark elevated surface, items separated by thin vertical lines, active tab "opens" into the page with matching background color. Tiny green top accent on active.

Concept 3 — Terminal Breadcrumb
Nav as a terminal path: ~/omar > home / projects / experience
No container at all. The nav reads like a terminal prompt. Active section is bright, others are dim. Monospace font, / separators, blinking cursor after active item. The build log hero flows directly into this.

Most unique and memorable. Directly ties to the build log terminal in the hero. The monospace / separators and blinking cursor scream "developer." Nobody else has this.
Concept 4 — Linear Flat Bar
Ultra-minimal: just text, nothing else
The Linear/Vercel/Tailwind approach. Zero decoration. Text links floating on the page background. No border, no container, no shadow. Active item is just brighter text with an optional tiny dot below.

Concept 5 — Command Palette Bar
Compact toolbar with | dividers and >_ prompt
Centered floating bar but reimagined as a command palette. Small border radius (6px not 24px), monospace, pipe dividers, >_ prompt icon. Feels like a miniature command bar, not a marketing pill.

My recommendation

Concept 3 (Terminal Breadcrumb) if you want the most unique, memorable nav that directly ties to the build log hero. Nobody has this.

Concept 1 (GitHub Underline) if you want proven, professional, immediately coherent with the diff view. This is what GitHub itself uses.

Either one would look great. The current pill is the only wrong answer.
