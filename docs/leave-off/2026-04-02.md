## Where We Left Off

The video-to-ASCII renderer is working — the Midjourney video plays and gets converted to ASCII characters in real-time with color preserved. **Two issues to fix next:**

1. **Video not filling viewport** — currently only rendering in a portion of the screen. Need to scale video sampling to cover full viewport (object-fit: cover equivalent for canvas sampling)
2. **Brightness threshold tuning** — may need to lower the cutoff so more of the image comes through, or boost the video brightness/contrast before sampling

## Key Design Decisions Made
- **Video-to-ASCII** is the technique for section backgrounds — not procedural noise, not static CSS
- **Midjourney video** as the source material for the languages world (hands typing)
- **Pink/purple/blue colorway** on dark is the target palette (from CÉLINE ARDENT reference)
- **Background should be bold** — fills majority of the page, not subtle accent decoration
- **Domain warp** is the backup plan if video ASCII doesn't work out
- Languages section = code editor section (not a separate skills card grid)
